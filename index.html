Emu Hunt

by: Jonathan Cornick

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make Emu Hunt because my class had decided on an emu theme for our games. After deciding on emu theme I immediately thought of the game Duck Hunt. After brainstorming other ideas a Duck Hunt style game seemed the most interesting.

2. What I Learned

One thing I learned from teachers and classmates was that the way my data like game time and bullets was set up was making it hard to play. I fixed this issue by moving these displays next to the target so that it is easier to see.

3. Architecture of My Game

Some different parts of the game that interact with each other are the objects: Target, Enemies, Explosions, and the ints score and bullets. These interact with each other in a way that allows you to kill enemies, draw explosions, and decrease or increase bullets and score using a checkCollisions method and using action preformed methods. The basic architecture of my game is that everything is run off of the runner class EmuHunt and the GamePanel class handles drawing and creating objects in the game. The ObjectManager checks for collisions and removes and adds objects that can be drawn. Other classes like Target and Enemy are used to create, draw, or track specific things like the Emus in the game and the target used to aim.

4. Technical Implementation

One particular aspect of my game that I'm proud of is the target/UI. I am proud of this part of the game because it was difficult at first to work around the target being offset. This challenge provided a learning opportunity because I had to do math and think about how things would look visually when I set their x and y position. Some steps I took to accomplish the goal of having a nice looking target that worked properly was having more clear code so I could find x, y, width, and height values to compare to on the target.

5. Most Challenging Part

The most challenging part of the project was working around my unorganized code and figuring out how to use update methods and separate classes effectively. I overcame this by asking my instructor, Mike, for help when things weren't working out properly. After this I realized some of my choices in organizing my code and choices in fixing problems were incorrect. I now have a better idea of how to organize code and the uses of multiple classes.

6. Future Fixes and Updates

If I was continuing working on this game one thing I would do would be to improve the hit-boxes of the enemies by adding multiple rectangles of different sizes as different hit-boxes. Another thing I would do would be to add an animation for the emus, I would do this by adding multiple alternating pictures and switch them on the emu.
